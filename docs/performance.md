# Звіт з лабораторної роботи №8: Профілювання та оптимізація

## 1. Опис виконаних робіт
- Налаштовано базові інструменти профілювання за допомогою модуля `cProfile`.
- Проведено аналіз продуктивності для двох сценаріїв: малого (5 осіб) та великого (1000 осіб).
- Виявлено та виправлено помилку `NameError (np)`, що виникала через відсутність бібліотеки NumPy.

## 2. Аналіз "гарячих точок" (Hot Spots)
Згідно з результатами профілювання:
- **Найповільніша функція:** `initialize_population` (займає ~50% часу виконання основного коду).
- **Інфраструктурні витрати:** Значна частина часу витрачається на операції введення-виведення (I/O) через активне логування.

## 3. Впроваджені оптимізації
- **Виправлення параметрів:** Мутація змінена з помилкового значення 99.0 на стабільне 0.05.
- **Алгоритмічна оптимізація:** Використано спискові включення (list comprehensions) для прискорення створення початкової популяції.

## 4. Висновок
Профілювання допомогло виявити неефективні ділянки коду та виправити критичні помилки валідації, що дозволило забезпечити стабільну роботу алгоритму при великих навантаженнях.
